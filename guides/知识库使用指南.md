# 知识库使用指南

## 概览

这是一个基于 Obsidian 的个人知识库，支持六种内容类型、命令行快速捕获，以及自动构建为静态网站供浏览器阅读。

### 核心工作流

```
快速捕获 (CLI/Obsidian) → 整理成篇 (Obsidian) → 发布到网站 (note publish)
```

---

## 目录结构：东西写哪

```
note/
├── inbox/          ← 📥 碎片想法，按日期一条，不纠结分类
├── daily/          ← 📅 日常记录，今天干了什么
├── notes/
│   ├── learning/   ← 📚 学习笔记（学完后的总结）
│   ├── ideas/      ← 💡 想法成篇（把模糊点子写成型）
│   └── thinking/   ← 🧠 深度思考
├── projects/       ← 🚧 项目设计与方案
├── weekly/         ← 📊 周报复盘
├── archive/        ← 📦 不再活跃的旧内容
├── guides/         ← 📖 使用教程
└── templates/      ← 🧩 模板文件（不要手动改）
```

### 判断规则

| 你的状态 | 放哪里 |
|---------|--------|
| 脑子里只有模糊想法，一两句话 | `inbox/` |
| 今天做了什么，流水账 | `daily/` |
| 学完一个东西，想整理输出 | `notes/learning/` |
| 有个想法想展开写写 | `notes/ideas/` |
| 想开始一个新项目 | `projects/<项目名>/` |
| 周末回顾 | `weekly/` |

## CLI 命令速查

脚本位置：`~/.local/bin/note`，已在 PATH 中，终端直接使用。

### note capture — 快速捕获碎片

```bash
note capture "用 Rust 重写那个 CLI 工具，性能应该会好很多"
```

自动追加到 `inbox/YYYY-MM-DD.md` 的「想法 💡」区域。不用管文件存不存在——不存在会自动创建。

每天可以多次 capture，所有碎片归到同一个每日文件。

### note daily — 今日日常记录

```bash
note daily
```

创建（或打开）今日的日常记录文件 `daily/YYYY-MM-DD.md`，并在 Obsidian 中打开。

模板包含四个区域：
- **做了** — 今天做了什么，流水账
- **值得记的** — 最有意思的一件事
- **心情** — 一个词或一句话

### note new — 从模板生成笔记

```bash
# 学习笔记
note new learning "Rust 生命周期"

# 想法笔记
note new idea "用 Tauri 写一个跨平台笔记 App"

# 项目设计
note new project "个人博客重构"
```

生成的文件位置：

| 类型 | 路径 |
|------|------|
| learning | `notes/learning/<标题-slug>.md` |
| idea | `notes/ideas/<标题-slug>.md` |
| project | `projects/<标题-slug>/<标题-slug>.md` |

每条命令会自动在 Obsidian 中打开新文件。文件名自动处理：中文保留，空格转 `-`，英文小写。

### note weekly — 周报

```bash
note weekly
```

自动生成当周周报 `weekly/YYYY-Www.md`，包含：
- 本周做了
- 值得记的
- 卡在哪
- 下周

### note publish — 发布到网站

```bash
note publish
```

执行三个步骤：
1. **构建**：Quartz 将 Markdown 编译为静态 HTML
2. **提交**：Git commit 所有变更
3. **推送**：Push 到 GitHub，GitHub Pages 自动更新

网站地址：`https://yuechuye.github.io/my-notes`

## 模板说明

所有模板采用**引导式提问**而非空白标题——按字段填写即可，不会「写了显空」。

每个模板有 `status` 字段追踪成熟度：

| 模板 | 文件 | status 流转 |
|------|------|-----------|
| 碎片捕获 | `capture.md` | —（临时内容，无 status） |
| 日常记录 | `daily.md` | —（日志，无 status） |
| 学习笔记 | `learning-note.md` | seedling → budding → evergreen |
| 想法笔记 | `idea-note.md` | seed → sprout → evergreen |
| 项目设计 | `project-design.md` | ideation → designing → building → shipped → archived |
| 周报 | `weekly-review.md` | —（按周归档） |

### 学习笔记的关键引导

- **一句话总结** — 用你自己的话说
- **核心概念** — 3-5 个关键点
- **为什么重要** — 和你已有知识的关联
- **实践** — 示例代码或操作步骤

### 项目设计的关键引导

- **动机** — 为什么想做
- **核心功能** — ≤3 个，防止膨胀
- **不做的事** — 故意砍掉的范围
- **下一步** — 这周的最小一步

## 跨设备使用

### 电脑端

- **编辑**：Obsidian 打开 `/Users/yuechu/MY/note/`
- **快速捕获**：终端用 `note` 命令
- **发布**：`note publish`

### 手机端

1. iPhone/iPad 安装 Obsidian App
2. 将 vault 放到 iCloud Drive
3. Obsidian App 打开 iCloud 中的 vault

> **注意：** 手机端可以直接编辑 Markdown、使用模板。但 CLI 脚本和 `note publish` 只在电脑端可用。

---

## 定期维护

- **每日**：有想法就 `note capture`，有产出就 `note daily`
- **每周**：`note weekly` 做周报；回顾 `inbox/`，有价值的整理到 `notes/`，没价值的删掉
- **学完东西后**：`note new learning` 输出一篇学习笔记
- **想到就记**：不要等到「想清楚再写」，碎片先进 inbox，之后慢慢整理

## 进阶定制

### 修改模板

模板文件在 `templates/` 目录，直接编辑即可。`{{date}}`、`{{title}}`、`{{week}}` 是变量占位符，`note new` 命令会自动替换。

### 修改网站风格

编辑 `quartz.config.yaml` 可调整：
- 站点标题
- 配色方案（亮色/暗色）
- 字体
- 忽略的目录

编辑后运行 `note publish` 生效。

### 添加新的笔记类型

1. 在 `templates/` 创建新模板
2. 在 `~/.local/bin/note` 的 `case "$type"` 中添加对应分支
